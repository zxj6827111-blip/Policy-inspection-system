# 上海区级政策文件安全巡检系统

本系统在 Windows 本机运行，通过真实 Chromium 浏览器串行访问政策公开页面。除上海市统一政策发布平台外，现支持普陀区政府网站的五个政策来源：区政府文件、委办局、街道镇、规范性文件、党政混合信息。系统检查七项详情表头、文号、日期、机构和政策解读/阅办联动/附件链接，并导出 Excel。

## 安全边界

- 全局并发固定为 1；页面和外链共用同一个限速器。
- 每次访问随机等待 5–10 秒，每 50 页强制休息 3–5 分钟。
- 同一地址最多重试 2 次，重试前分别等待 30–60 秒、2–5 分钟。
- 遇到 403、429、验证码、访问频繁提示或连续三次网络失败立即暂停。
- 单次最长运行 4 小时，之后至少冷却 30 分钟并等待手动恢复。
- 详情点击、翻页、区县筛选、政策解读、阅办联动和附件共用同一个限速器。
- 外链逐跳检查重定向，拒绝本机、内网、链路本地地址和异常端口，避免 SSRF。
- 不伪装浏览器指纹、不绕过验证码、不使用代理池。

## 安装与启动

在 PowerShell 中运行：

```powershell
.\setup.ps1
.\run.ps1
```

打开 `http://127.0.0.1:8765`。界面默认选中普陀区网站的五个政策来源，且“本次最多文件数”为 `0`，表示扫描该来源下的全部列表条目。

## 使用流程

1. 选择要检查的来源。页面中“栏目 ID”仅用于网站接口筛选，不代表文件数量；系统在任务开始时读取每个来源的实际分页总量。
2. 首次选择“全量建立基线”并保持条目上限为 `0`。只有状态为“已完成”且覆盖完整的全量任务可成为基线。
3. 后续选择“基于全量记录的增量扫描”，从“基准全量扫描记录”下拉框选定任务。系统自动锁定为这条基线完全相同的来源集合。
4. 点击“开始扫描”，观察当前 URL、覆盖数、表头完整/缺字段/无表头 PASS/基线复用统计和风控提示。
5. 可随时暂停、停止；异常暂停后，检查原因并等待冷却再恢复。
6. 从当前任务或历史任务下载 Excel。

限量任务达到本批上限后显示“部分完成”，不会伪装成全量完成。点击恢复会继续原区县、页码和条目位置，并继续使用创建任务时的每批上限。

增量扫描只与用户选择的全量基线逐条比较。URL、标题和列表发布日期均未变化的完整表头文件会复用基线结果；基线表头不完整、列表信息变化或新增条目会重新打开详情。无表头条目保持 PASS；同一 URL 出现在多个来源时只访问一次详情，但每个来源都会保留独立 Excel 行。

## 规则口径

- 页面文号和正文文号采用统一括号、空格和编号格式后比较。
- 普陀区网站详情页仅在检测到七项表头（索引号、主题分类、公开属性、成文日期、发文字号、发布日期、公开主体）后进行字段检查。无表头条目 PASS；任一字段缺失或日期格式无效生成一条 `META-001` 元数据问题。
- 文号年份与成文年份不一致列为文号问题。
- 工作日使用 `chinese-calendar` 提供的国务院节假日和调休口径；超出依赖支持年份会使任务失败并明确记录，不静默退化为只排除周末。
- 成文日至发布日期超过20个工作日，或发布日期早于成文日期，列为日期问题。
- 发文机构和文号机关代字不一致只标记为待人工复核。
- 标题主体、页面发文机构和正文落款存在规则可识别的差异时，也只标记为“待复核”，由控制页人工确认或排除。
- PDF和办公附件只检查可访问、可下载，不解析正文。

## Excel 输出

每个任务导出七张工作表：`问题汇总`、`超期与日期问题`、`文号与机构问题`、`外链问题`、`全量明细`、`元数据问题`、`扫描运行记录`。`全量明细`以每个来源的每个列表条目为一行，包含七项表头、缺失字段、处理结果、基线任务 ID 和链接；日期使用 Excel 日期类型，链接可点击，问题行有颜色标识。

字段风格参考样表，包含栏目名称、文件ID、标题、成文日期、发布日期、工作日差、时效状态、文号拆分、文号来源、问题类型、问题详情和链接。

## 数据与备份

运行数据位于 `data/policy_inspector.db`，Excel位于 `data/exports/`。数据库保存任务、分区进度、文件快照、规则结果、链接结果、人工复核和安全事件。

备份步骤：

1. 在控制页停止当前任务并关闭服务窗口。
2. 确认 `8765` 端口已不再监听。
3. 复制整个 `data/` 目录到带日期的备份目录。

恢复步骤：

1. 关闭服务，将现有 `data/` 另行备份。
2. 用目标备份中的 `data/` 替换当前目录。
3. 重新运行 `run.ps1`；系统会自动执行只增不删的数据库兼容迁移。
4. 打开历史任务核对区县、游标和覆盖数量后，再手动恢复未完成任务。

不要在服务运行时直接编辑或只复制 SQLite 主文件，以免遗漏 WAL 中的数据。

## 风控处置

- `403/429/验证码`：不要立即恢复；系统会阻止冷却结束前恢复，之后仍应先在普通浏览器人工确认网站是否正常。
- 页面结构变化：任务会以“采集器需要更新”暂停，禁止通过提高重试频率处理。
- 连续网络失败：先检查本机网络和目标站状态，再手动恢复。
- `robots.txt` 明确禁止目标路径：系统停止访问，该路径改为人工处理。
